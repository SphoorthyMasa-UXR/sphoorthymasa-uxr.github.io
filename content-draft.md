## Case Study 1: Guide — Feature Evaluation

**Hero Stat:** ~$7.46M estimated annual value impact

**Problem (1 line):**
44% of guide sessions ended without playback. Users couldn't find content on the most-used feature.

**My Role:**
Led end-to-end: stakeholder alignment → quant analysis → user interviews → synthesis → design collaboration → usability testing → shipped.

**Process:** *(visual diagram — not text)*
Stakeholder Interviews → Data Analysis → Scope Definition → Generative Interviews → Synthesis → Design Collaboration → Usability Testing → Ship

**Key Findings (3 cards):**

| Finding | Data |
|---------|------|
| Filters are invisible | 72% used default "ALL" view. Only 9% used Favorites — yet those sessions had 63.9% playback (8.5% higher) |
| Channel clutter overwhelms | Users described "scrolling for eternity." Cognitive overload drives exits within 20 seconds |
| Fear blocks personalization | Users avoid changing settings because they fear disrupting their workflow |

**Impact Metrics:**

| Metric | Result |
|--------|--------|
| Filter usage | ↑ 39% |
| Favorites engagement | ↑ 57% |
| Launch time | 25s → 17s |
| CX "how do I use this" tickets | ↓ 31% |
| UMUX-Lite Score | 92 |
| Estimated annual value | ~$7.46M |

**One-liner takeaway:**
Research-driven redesign turned a broken discovery experience into the most efficient content-launch feature in the product.

## Case Study 2: Store-to-Door — Strategic Demand Research

**Hero Stat:** n=539 | 4 hypotheses tested | Budget reallocation recommended

**Problem (1 line):**
Leadership assumed older, rural, tech-challenged users would love in-home phone setup. They needed data before spending.

**My Role:**
Designed the full study end-to-end: hypotheses, survey instrument, quota strategy, fielding, statistical analysis, and VP-level presentation. Solo researcher.

**Process:** *(visual diagram)*
Hypothesis Formation → Study Design → Fielding (2 days) → Statistical Analysis → Insight Synthesis → Stakeholder Presentation → Budget Decision

**Key Findings (3 cards):**

| Finding | Data |
|---------|------|
| H1 DEBUNKED: "Older users who struggle with tech will prefer this" | Correlation ≈ 0. Interest driven by convenience, not inability. t=7.22, p<0.001 |
| H2 NOT SUPPORTED: "Rural users prefer it due to no stores" | Urban trended higher. p=0.12, not significant |
| H3 CONFIRMED: Active switchers love it | 18-34 highest interest (47%), highest switch intent (29%), highest NPS (7.7/10) |

**Impact Metrics:**

| Metric | Result |
|--------|--------|
| Budget recommendation | Reallocate 70%+ to 18-44 |
| Conversion potential | 3-5x higher ROI vs. 55+ |
| Winning message identified | "Free delivery + free setup. Just pick a time." |
| Alternative service designed | Video-call setup for 55+ |

**One-liner takeaway:**
Data debunked the core business assumption, preventing misspent marketing budget and redirecting investment to 3-5x higher-ROI segments.

## Case Study 3: IMEI Checker — Rapid Optimization

**Hero Stat:** 3-day study | n=1,000 survey + n=15 usability | 91K users/month impacted

**Problem (1 line):**
4,363 users/month dropped off at the IMEI entry step despite a redesign. Nobody knew why.

**My Role:**
Designed and ran parallel quant (n=1000) + qual (n=15) in 3 days. Synthesized, presented recommendations, partnered with Product and E-commerce to ship.

**Process:** *(visual diagram)*
Problem Identification → Parallel Study Design → Survey (n=1000) + Usability (n=15) → Synthesis → Recommendations → Implementation

**Key Findings (3 cards):**

| Finding | Data |
|---------|------|
| Users don't know what IMEI is | Low baseline awareness. Cognitive friction before any action |
| Trust > Instructions | 56.4% would enter IMEI if "why it's needed" is explained. The issue isn't *how*, it's *why* |
| Help link breaks the flow | IMEI help redirects to generic BYOD page. Users exit funnel and don't return |

**Impact Metrics:**

| Metric | Result |
|--------|--------|
| Users affected | 91K+ monthly |
| Previous bounce rate | 4.79% at critical step |
| Solution | Education-first + inline help + trust messaging |
| Turnaround | 3 days from brief to recommendations |
| Post-launch | Scaling to all IMEI-dependent flows |

**One-liner takeaway:**
Rapid research revealed the barrier wasn't usability — it was understanding and trust. Fixing the "why" before the "how" kept users in the funnel.
